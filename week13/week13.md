state 로도 관리는 가능하나 큰프로젝트일수록 리덕스 사용이 관리에 용이

여러 컴포넌트가 동일한 상태를 공유해야 할 때 유용, 실제로 업데이트가 필요한 컴포넌트만 리렌더링 하는 최적화도 가능

리액트 프로젝트 생성, Prettier를 적용해 일관된 코드 스타일
리액트 프로젝트에서 리덕스 사용시 프레젠ㅔ이셔널 컴포넌트와 컨테이너 컴포넌트를 분리하는 패턴이 가장 많음

*프레젠테이셔널 컴포넌트: 상태를 관리하지 않고 props만 받아 화면을 렌더링 하는 컴포넌트
*컨테이너 컴포넌트: 리덕스와 연동되어 상태를 받고, 리덕스 스토어에 액션을 디스패치하는 컴포넌트

리덕스 사용시 액션, 액션 생성,리듀서 코드 작성 필요
